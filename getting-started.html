Getting Started

Background

The new ASU web standards started with a reference implementation from asu-ws.29thdrive.com. WebSpark, ASU slimspark, and the ASU Web Standards Bootstrap Addon all are different implementations based on this 29thdrive reference. The ASU Web Standards Bootstrap Addon is intended to be a universal source for the ASU web standards that can be applied to any website, reguardless of what technology was used to build the site.

The new ASU Web Standards is are available on the following platforms:

  • Wordpress Theme - Built using the ASU Web Standard Bootstrap Addon
  • WebSpark - Drupal (7) Distribution
  • ASU slimspark - Drupal (7) Theme
  • Other/Custom Website - ASU Web Standards Bootstrap Addon

Wordpress Theme

If you are creating a Wordpress website, we recommend using the Wordpress Theme to integrate the web standards into your website. If you need to make customizations or additions to this theme for your application, create a child theme so that future updates are possible.

The ASU Web Standards Wordpress Theme is available through GitHub. If you are starting from scratch, simply add it to themes directory of your Wordpress installation, or install the theme through the admin panel. Make sure to activate the theme through Themes section of your admin panel and your website will be using the Web Standards.

If you are migrating a Wordpress installation to use the new Web Standards, you may need to fix content on your website. Check out the Migration Documentation for a more thorough explanation of how to migrate your Wordpress installation.

Download the Wordpress Theme

WebSpark

If you are creating a new Drupal website, Webspark has the new web standards are already integrated for you.

WebSpark is managed by UTO, a Drupal 7 Distribution that has its own implementation of the new ASU Web Standards, it does not use the ASU Boostrap Addon.

Read More: Webspark

ASU slimspark

A Drupal 7 Theme maintained by UTO. Intended for when the full WebSpark Disturbution isn't a good fit for an existing drupal site.

Read More: ASU slimspark

ASU Web Standards Bootstrap Addon

The ASU Web Standards Bootstrap Addon is available through GitHub. The raw JavaScript, raw SCSS files, and the current CSS, JS, and Font builds are available in the repository.

If you are creating a custom website or a theme to another content management system, we recommend using the compiled JS and CSS files directly.

Download the Web Standards Bootstrap Addon

Using the Web Standards

The Web Standards are built on top of Bootstrap and require the Google Font "Roboto". The following is an example of what your head tag might look like: 

<link href='//fonts.googleapis.com/css?family=Roboto:900,400,300,700,500' rel='stylesheet' type='text/css' /><!-- The main app css file -->
<link href="./css/bootstrap.css" rel="stylesheet" type="text/css" />
<link href="./css/bootstrap-asu.css" rel="stylesheet" type="text/css" />
<link href="./css/bootstrap-asu-theme-base.css" rel="stylesheet" type="text/css" />

bootstrap.css would be the latest version of Twitter's Bootstrap.

bootstrap-asu.css is the latest build from the GitHub Repo of the ASU Web Standards.

bootstrap-asu-theme-base.css is an initial set of CSS that themes the your website to the default ASU theme. This file can be customized by editing the SCSS files that are found in /scss/custom in the GitHub repo and compiling them.

Your the code right before your closing body tag might look like this: 

<script src="./js/jquery.js"></script>
<script src="./js/bootstrap.js"></script>
<script src="./js/bootstrap-asu.js"></script>
Watch Out! The Web Standards require Bootstrap to work properly.
Watch Out! Bootstrap requires jQuery for full functionality.

Structure

Once downloaded, you will need the following files from the Github repository: 

build/
├── css/
|   ├── bootstrap-asu.css
|   └── bootstrap-asu-theme-base.css
├── js/
│   ├── bootstrap-asu.js
└── fonts/
    ├── fontawesome-webfont.eot
    ├── fontawesome-webfont.svg
    ├── fontawesome-webfont.ttf
    ├── fontawesome-webfont.woff
    └── FontAwesome.otf

This is the most basic form of the Web Standards. They are precompiled files for quick drop-in usage of any project that uses Bootstrap. We provide CSS, JS, and font files for convenience. Fonts from FontAwesome are included.

When integrating the Web Standards into a Bootstrap project, you will get the following directory structure:

webroot/
├── css/
│   ├── bootstrap.css
│   ├── bootstrap-asu.css
│   ├── bootstrap-asu-theme-base.css
├── js/
│   ├── jquery.js
│   ├── bootstrap.js
│   ├── bootstrap-asu.js
└── fonts/
    ├── fontawesome-webfont.eot
    ├── fontawesome-webfont.svg
    ├── fontawesome-webfont.ttf
    ├── fontawesome-webfont.woff
    └── FontAwesome.otf

Theming

Theming is done by creating your own boostrap-asu-theme-base.css. This CSS file controls only the colors that appear on your website.

The SASS files that are used to create a theme base are in scss/custom. The _variable.scss file contains all of the color codes used to theme an ASU website.

In order to compile your SASS files into a CSS file, use a SASS processor to compile bootstrap-asu-theme-base.scss. This will create a custom version of boostrap-asu-theme-base.css that you can use on your website.
